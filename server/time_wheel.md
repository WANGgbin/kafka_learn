在 kafka 中有很多的延时任务, 即延迟一定时间才开始执行的任务. 那么 kafka 中的延时任务是怎么实现的呢?

通过时间轮.

# 时间轮
- 如何添加延时任务?

我们可以定义一层时间轮有多少格, 以及每个格代表的时间跨度. 除此之外, 我们还可以定义多层时间轮, 这样可以表示更大的时间跨度.

当我们要添加一个延时任务的时候, 需要通过延时时间以及时间轮当前的刻度(注意不是当前时间,时间轮的刻度是由延时任务的运行推进的.)计算出在第几层时间轮以及第几格, 然后加入到对应的链表中即可(时间轮的每一格都是一个双向链表, 将所有要执行的任务串联在一起.)

为什么不直接使用 DelayQueue 添加任务呢? 主要是 DelayQueue 底层实现是优先级队列,添加元素的时间复杂度是 O(nlogn), 而通过时间轮添加任务绝大多数都是 O(1), 如果需要将时间轮格子对应的链表元素添加到 DelayQueue(只有当前任务是格子里的第一个任务的时候,才会触发此操作), 则时间复杂度还是 O(nlogn)

- 怎么判断延时任务到期?

kafka 通过 java 的 DelayQueue(底层是个优先级队列) 来执行延时任务. DelayQueue 的工作原理是消费线程会判断当前队列中队首元素的时间戳跟当前时间的 diff, 如果小于等于, 则执行对应的任务. 否则, 消费线程会睡眠一定的时间. 直到睡眠时间到或者队里插入了比当前队首元素时间戳更小的任务的时候, 会重新激活消费线程.

kafka 中添加延时任务的时候, 如果新任务对应的时间轮的小格还没有添加到 DelayQueue 中, 则会往 DelayQueue 中添加一个元素.

- 时间轮是怎么推进的?

当 DelayQueue 中的任务到期后, 消费线程执行任务. 并同时通过将时间轮的当前刻度调整为 当前执行任务对应的时间, 来推进时间轮.
